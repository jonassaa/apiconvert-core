# SemVer Lockstep

Repository, package, and schema versions are intentionally lockstep.

- Git tag: `vX.Y.Z`
- NuGet package: `X.Y.Z`
- npm package: `X.Y.Z`
- Rules schema: `X.Y.Z`

## Why lockstep exists

- Clear compatibility expectations for adopters
- Simpler release communication
- Predictable parity testing and support workflows
